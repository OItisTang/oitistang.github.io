“宇言”：使万物沟通与协作，创无限可能

"Universa": Make Everything to Communicate and Cooperate, Create Infinite Possibilities

“宇言”，即Universa，由两部分组成：
  1. The Universa Language：它是一个通信协议，规范万物都具有统一接口，从而万物可以相互沟通，使得万物间协作成为可能。
  2. The Universa Cloud：它是对The Universa Language协议的云端部分具体实现。
    它作为通信基础设施为万物提供沟通通路；
    它作为运算框架根据The Universa Language协议为所有连接于此的事物创建一个统一呈现，这样的呈现将包含属性、事件、方法，并以此创建编程和运算环境；
    它作为应用平台允许我们利用万物沟通接口编写应用程序，并帮助我们执行这些这些应用程序。
通过“宇言”，万物得以沟通，协作成为可能。世间万物的数量巨大，任何两个事物都可以沟通与协作，这种可能性是呈指数级增长的。
无限可能性创造着新的文明，而这种文明并非是文明的一次简单增长，是文明的二次方。

The Universa Language：万物沟通的协议

The Universa Language: The Protocol for Everyting to Communicate

The Universa Language协议（简称“宇言”协议），
是一种交互规范，它要求所有事物都具有统一接口。
是一种语义协议，它定义交互的流程与交互内容的语法格式。

所有符合“宇言”协议的事物都将成为一个具有统一接口的对象，
“宇言”协议将万物的状态、行为和能力分别抽象成属性、事件和方法。
随着科技的发展，我们每个人都随身携带一个或多个智能设备，如iPhone、智能手环等。这些设备作为一个代理早就把我们变成Internet中的一部分。
因此这些设备可以进一步为我们人实现属性、事件和方法，从而我们人除了作为人类存在以外，同时在另一个角度也变成了符合“宇言”协议的事物。

因此，所有这些事物，包括人，可以通过触发应用程序，再由应用程序执行其他事物的方法，从而实现万物协作。


“宇言”协议的典型交互流程：
1、将事物连接到The Universa Cloud
我们告诉Cloud连接事物的必要信息（如IP、密码等），Cloud会请求连接事物，
事物回复一份自我描述给Cloud，
Cloud以此为事物创建呈现，这样的呈现将包含属性、事件、方法。
2、事件处理
当事物内部状态发生变迁，它会发送一个事件给Cloud，
Cloud根据事物的呈现来判定如何处理这个事件。
比如可以用事件所包含的值来更新这个事件的某一个属性值。
该事件还可以作为执行其他任务的触发条件。
3、读取事物的属性值
当我们想去读取事物的某个属性值时，
Cloud会帮助我们查询这个事物的呈现，
并从中获取最新的值返回给我们，而无需去直接跟事物获取。
这样的好处是，Cloud为事物本身屏蔽了访问压力，如果事物本身能力较弱，它根本不用担心因负载过大而宕机。
4、执行方法
当我们想执行事物的某个方法时，
Cloud会查询该事物的呈现，并以此要求我们输入所需的参数，
为我们发送一条执行方法的请求报文，
事物接到该报文后执行相应动作，并返回一定的值给Cloud告知成功与否。
5、事物通过应用程序进行协作
如前所述，Universa是一个协作平台，我们可以对事物的属性、事件、方法编写一个应用程序并保存于Cloud。
该应用程序可以声明某一个事物发来的某一个事件满足什么样的条件时，该应用程序得以运行，它可以去执行另一个事物的一个方法。
触发条件可以是多个，执行的动作也可以是多个。

“宇言”协议的语法格式：
1、“连接”报文
当我们告诉Cloud我们想要连接的事物的IP、端口和密码后，Cloud将产生一个连接请求报文发给事物。
{
            “type” : “connect”,
            “secureKey” : “ABCDEF”
}
2、“自我描述”报文
当事物收到连接报文以后，如果该事物同意连接，那么它将产生一个“自我描述”的报文，
该报文主要告诉Cloud自己拥有多少个属性及其类型、多少个事件及其携带值列表声明、多少个方法及其参数列表声明等。
当Cloud收到该报文，就可以根据这些描述为该事物创建一个呈现。
{
            “type” : “selfDescription”,
            “eventPort” : 9001,
            “property” : [
                        {
                                    “name” : “property1”,
                                    “type” : “number”,
                                    <other optional fields>
                        },
                        <other properties>
            ],
            “event” : [
                        {
                                    “name” : “event1”,
                                    “type” : “number”,
                                    “updateProperty” : “property1”
                        },
                        <other events>
            ],
            “function” : [
                        {
                                    “name” : “function1”,
                                    “type” : “boolean”,
                                    “parameter” : [
                                                {
                                                            “name” : “parameter1”,
                                                            “type” : “number”,
                                                            <other optional fields>
                                                },
                                                <other parameters>
                                    ]
                        },
                        <other functions>
            ]
}
3、“事件”报文
当事物内部发生状态变迁时，它会发送一个事件报文并携带值给Cloud。
Cloud可依此更新该事物的属性值或者运行应用程序。
{
            “type” : “event”,
            “event” : “event-name”,
            “value” : <value of the event>
}
4、“方法执行”报文、“方法响应”报文
当我们通过Cloud请求执行一个事物的方法时，Cloud会创建一个方法执行请求报文，其中包含方法名称和相应的参数列表。
{
            “type” : “call”,
            “function” : “function-name”,
            “parameter” : {
                        “parameter1” : <value1>,
                        “parameter2” : <value2>,
                        <other parameter-value pairs>
            }
}
事物执行相应动作之后，返回一个报文并携带一定值来告知Cloud执行结果。
{
            “type” : “function”,
            “function” : “function-name”
            “value” : <value>
}
5、“断开连接”报文、“再见”报文
当我们希望从Cloud中移除一个事物，Cloud会发送一个报文给事物请求断开连接。
{
            “type” : “disconnect”
}
如果事物同意，那么它就会返回一个同意断开连接的报文和Cloud说再见。
{
            “type” : “byebye”
}
6、“错误”报文
任何时候，事物不能正确处理Cloud发来的请求时，就会返回一个错误报文，包含错误信息。
{
            “type” : “error”,
            “message” : “error message”
}

“宇言”协议语义部分是基于JSON的，JSON轻量、低冗余、全语言支持、易扩展。
“宇言”协议的传输部分是基于TCP或UDP的，但不限于此。一个特殊的自治“文明二次方”系统中，可以有自己独特的实现。

当我们通过Cloud请求执行一个事物的方法时，Cloud会创建一个方法执行请求报文，其中包含方法名称和相应的参数列表。
{
            “type” : “call”,
            “function” : “function-name”,
            “parameter” : {
                        “parameter1” : <value1>,
                        “parameter2” : <value2>,
                        <other parameter-value pairs>
            }
}
事物执行相应动作之后，返回一个报文并携带一定值来告知Cloud执行结果。
{
            “type” : “function”,
            “function” : “function-name”
            “value” : <value>
}
5、“断开连接”报文、“再见”报文
当我们希望从Cloud中移除一个事物，Cloud会发送一个报文给事物请求断开连接。
{
            “type” : “disconnect”
}
如果事物同意，那么它就会返回一个同意断开连接的报文和Cloud说再见。
{
            “type” : “byebye”
}
6、“错误”报文
任何时候，事物不能正确处理Cloud发来的请求时，就会返回一个错误报文，包含错误信息。
{
            “type” : “error”,
            “message” : “error message”
}

“宇言”协议语义部分是基于JSON的，JSON轻量、低冗余、全语言支持、易扩展。
“宇言”协议的传输部分是基于TCP或UDP的，但不限于此。一个特殊的自治“文明二次方”系统中，可以有自己独特的实现。

The Universa Cloud：一个强大的基础设施、运算框架、应用平台

The Universa Cloud: A Powerful Infrastructure, Computing Framework, Application Platform

The Universa Cloud是“宇言”的重要组成部分和核心驱动力。
它主要由四大功能模块组成：
  1. Universa Protocol Module：The Universa Cloud是“宇言”协议的云端实现部分。该功能模块主要完成与事物间的通信。
    事物连接到Cloud之后，该模块会为事物创建一个呈现，该呈现包含属性、事件、方法。
    当事物发来事件时，该模块会对事件进行处理。
  2. Web Server：我们无法与事物直接进行交互和对其进行管理，因此我们需要一个图形化用户界面，通过Cloud作为代理完成这些操作。
    首先该模块为我们提供用户账户管理功能。其次该模块会为事物的呈现提供图形化交互界面，通过Web的方式展示出来，使得事物可以看得见、摸得着、用得了。
    该模块还对事物的属性值进行图形化，使得我们可以通过图表的方式去审视事物的属性值（当前值和历史值）。
    该模块还会为事物创建RESTful API，使得事物可以和现有Internet应用无缝整合。
    出于安全考虑，该模块还会对事物及其API进行访问控制。最后该模块还允许我们创建应用程序以便按需执行。
  3. Application Platform：该模块提供对事物的编程环境，还会接收到Universa Protocol Module发来的事件通知，
    并解析存储于Cloud的应用程序，当该事件满足某个应用程序的触发条件时，执行该应用程序。
  4. Database：该模块是其他三个模块的服务模块，它会存储事物的呈现、事物的属性值、应用程序、用户账户、访问控制数据、Web服务器自身数据等。

2、为事物创建统一呈现
事物是现实世界的实体，当我们需要和事物沟通（或者事物间需要相互沟通）、管理事物的时候，
我们不希望直接去和事物对话（实际上大部分时候不太可能），
也不希望直接动手去管理这些具体的事物（原因可能是太远、太重、太大、太小、太多等等）。
“宇言”协议将所有事物都抽象为具有统一接口的对象，因此所有相互沟通与协作是通过这些虚拟对象来完成的。
而The Universa Cloud就会为事物创建这种虚拟呈现。
这个创建过程是全自动的，我们无需干预，只需告诉Cloud必要连接信息，Cloud就会主动去联系事物，
从事物那里获得它的“自我描述”，根据这个“自我描述”自动创建一个呈现。
这一切得益于“宇言”协议。

3、事物的事件处理
事物主动推送事件将解决两个问题。
第一，如何保持事物的属性值始终最新？
如果我们以主动拉取（pull）的方式获取，那么无论我们设置多长的时间间隔都不能保证属性值始终最新。并且过多查询请求是对资源的一种浪费。
而事物本身是最清楚什么时候需要更新属性值的——内部状态发生变迁的时候，这时它发送一个事件给Cloud并携带合适的数据值，
这样Cloud通过查询该事物的呈现得知该事件可以更新某个属性值，Cloud就会帮忙保持属性值更新。
第二，我们希望在某个事物发生某些变化的时候采取一些对策。
事件是最好的通知，Cloud收到这个事件后，就允许我们执行预先编写好的应用程序，从而实现我们采取的对策。

4、图形化用户界面
我们需要和事物沟通、编写应用程序、实现事物间协作。

这样的好处是：

无需安装任何额外的软件，所有流行操作系统都自带浏览器。
其次，当前Internet中有许多成功的应用如SNS等，这套图形界面既可以利用现有Web应用实现自己，
又可以把自己作为对现有Web的扩展，从而融入现有Internet，帮助现有Internet平滑进入下一阶段。

我们编写的应用程序也会保存在我们的账户里面，还有事物的属性值及其历史等。

我们还可以把这个URL发布到社交网站上去和好友分享。

Cloud提供若干方式让我们以图形化的形式去查看事物的属性值或者审视属性值的历史变迁。

8、为事物创建RESTful API
读取事物的属性值需要点击若干链接？
执行事物的方法需要填写一个表单？
这些对我们人来说是直观而自然的交互方式。
Universa不是一个独立个体，它无缝融入现有Web。
一些现有的Web应用通过RESTful API的形式相互合作，
而Cloud为满足需求，根据事物的呈现自动为事物创建RESTful API。

9、访问控制
安全问题始终很重要。
除了通过用户账户管理来控制其他用户不能直接管理我们的事物之外，
我们分享出去事物的URL是为了让别人也来使用我们的事物，
那么这里就需要更细腻的访问控制了。
Cloud提供对每个事物访问请求和控制。
好友第一次访问我们的事物时，是需要发送请求给我们的，我们批准之后，他们才可以继续使用。
如果需要，Cloud可以提供细致到单个属性、事件、方法的访问控制。

10、应用程序创建
为了实现事物间协作，我们需要编写应用程序。
Cloud为我们提供在线编写应用程序的功能。
它甚至是可视化的，左边是哪些事物的事件及其需要满足的条件，右边是相应执行的方法和参数。
点击保存，Cloud就会帮我们存储这个应用程序。

11、应用程序执行
Cloud会处理事务发来的事件，当该事件满足某个应用程序的运行条件时，
Cloud就会唤起该应用程序，执行该应用程序指定的所有动作。

Alice拥有一个符合“宇言”协议的智能温度计Thermometer。她希望把它连接到The Universa Cloud上。

我们可以在这个页面中很清楚地看出这个Thermometer拥有多少属性、事件、方法。

Alice可以通过表盘图的形式查看当前温度计的温度值。也可以通过曲线图的形式查看温度计温度值历史。

Universa在生活中的应用案例：
周末快到了，我在手机的日历中周六那一天写下“邀请三个好友到家里来做客”。
手机发送这个事件到Universa Cloud中。
之前购买并部署的Smart Party Planning应用被触发运行。
该应用调用我的淘宝账户去花店预定了周六的两束花朵。
该应用询问我的冰箱里还有多少水果。然后计算得到还需要一斤苹果、两斤香蕉。
于是该应用向水果店订购了这些水果。
水果派送员上门时，他的胸牌被我家大门识别为可信伙伴，因此大门打开窗口允许派送员把水果递进去。
家里的管家机器人把水果送进冰箱。
周六到了，Famous Cook应用根据我的口味从Amazing Recipe网站下载了几个有名菜单，
推送给我的智能厨房，不一会儿，几个香喷喷的菜就做好了。
两束花插在花瓶里了，水果也由智能水果机削好、切好了。
Party正式开始！生活就是这么简单美好！

Universa在工作中的应用案例：
公司的大门识别员工（可以通过胸卡，或者探测体温、心跳等生理特征），允许其进入，并自动记录上下班时间。
如有访客近来，自动派发门卡给访客，访客所有的行为和路径被跟踪。
所有物品的出入皆由大门记录，如有异常，自动报警和关门。
A项目组6个人准备开一个会议，Meeting Planner应用自动找到一个合适大小的会议室，打开投影仪，
并开启一个WebEx在线会议，同时触发几位同事的WebEx客户端进入会议。
在会议中，几位同事讨论并编写了一个应用，它控制实验室设备迅速组成试验网络。
下班了，Power Saver应用根据实验室设备的负载情况自动调度这些设备在合适的时间关机。

Universa在科研中的应用案例：
无数花草树木、飞禽走兽、高山流水都无时无刻不在向我们诉说着它们的感受，
我们可以由此获得大量来自自然的数据，进行分析，发现规律。
自然万物还是我们进行科学实验的直接试验场，能让我们知道人类行为是如何影响到周围环境的。
指导我们更好地利用自然、和自然共存、保护自然。

Universa在娱乐中的应用案例：
想像梁朝伟那样，有时闲着闷了，会临时中午去机场，随便赶上哪班就搭上哪班机，
比如飞到伦敦，独自蹲在广场上喂一下午鸽子，不发一语，当晚再飞回香港，当没事发生过一样吗？
不，你不需要坐飞机。
只需去一个超级模拟屋，带上虚拟现实眼镜。
你想去英国伦敦广场，你看到的就是伦敦广场上摄像机给你发回的画面，听到的就是那里话筒接收到的鸽子叫声。
你想去巴西里约热内卢海滩，你就可以晒到根据海滩上阳光探测器的数据模拟的阳光。
你想去法国普罗旺斯薰衣草田，你就可以闻到根据探测到的花香成分调配的薰衣草花香。
这个模拟屋可以实时模拟当地的风向、温度、适度、声音，让你身临其境。
